knitkit is a tiny (< 5 KB), zero-dependency, **runtime-first** module federation system built on
native ES modules and import maps. The same manifest drives the browser, Node SSR, and the edge.

<Note>
  Pre-1.0, published under the `@knitkit/*` npm scope. APIs may change before 1.0.
</Note>

## Why knitkit

- **No bundler plugin.** A runtime, a manifest, and a small CLI. Externalize shared deps in one line.
- **Singletons by construction.** One import-map entry per shared package → one instance, shared by the host and every remote via the browser module cache. The "more than one copy of React" bug class is gone by design.
- **Debuggable.** Open DevTools, read the import map, see which version won and from where.
- **Everywhere.** Browser, Node SSR (`module.register` + SRI), and edge composition — one manifest.
- **Small.** Federation overhead is ~3.5 KB brotli; [measured ~7–8× smaller](/vs-module-federation) than Module Federation.

## Quick taste

```html
<script type="importmap">
  { "imports": { "@knitkit/runtime": "/runtime/index.js", "react": "https://esm.sh/react@18.3.1" } }
</script>
<script type="module">
  import { registerRemotes, loadRemote } from "@knitkit/runtime";
  await registerRemotes([{ name: "checkout", manifest: "/federation/knit.manifest.json" }]);
  const CartWidget = await loadRemote("checkout/CartWidget");
</script>
```
